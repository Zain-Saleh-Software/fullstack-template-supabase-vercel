# i18n Skill (Vercel + Supabase Stack)

**Description:** Patterns for internationalization using `next-intl` with locale routing and SSR.
**Role:** The Executor

## 1. Setup

`next-intl` is configured with middleware-based locale routing in `src/middleware.ts`.

### Supported Locales

Defined in `src/i18n/request.ts` and `src/i18n/routing.ts`:

```typescript
export const locales = ['en', 'ar'] as const;
export const defaultLocale = 'en';
```

## 2. Adding Translations

Messages live in `messages/` directory as JSON files:

```json
// messages/en.json
{
  "accounts": {
    "title": "Accounts",
    "create": "Create Account",
    "edit": "Edit Account",
    "delete": "Delete Account"
  },
  "common": {
    "save": "Save",
    "cancel": "Cancel",
    "loading": "Loading..."
  }
}
```

```json
// messages/ar.json
{
  "accounts": {
    "title": "الحسابات",
    "create": "إنشاء حساب",
    "edit": "تعديل الحساب",
    "delete": "حذف الحساب"
  },
  "common": {
    "save": "حفظ",
    "cancel": "إلغاء",
    "loading": "جارٍ التحميل..."
  }
}
```

## 3. Using Translations

### In Components

```typescript
import { useTranslations } from 'next-intl';

function AccountsPage() {
  const t = useTranslations('accounts');
  return <h1>{t('title')}</h1>;
}
```

### In Server Components

```typescript
import { getTranslations } from 'next-intl/server';

export default async function AccountsPage() {
  const t = await getTranslations('accounts');
  return <h1>{t('title')}</h1>;
}
```

## 4. Locale Routing

- URLs follow `/en/accounts`, `/ar/accounts` pattern
- `next-intl` middleware automatically detects and redirects
- The locale is accessible via `params.locale` in page components

## 5. RTL Support

For Arabic (`ar`) and other RTL languages:

```tsx
// Use dir="rtl" on the HTML element based on locale
<html dir={locale === 'ar' ? 'rtl' : 'ltr'}>
```
